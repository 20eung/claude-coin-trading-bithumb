# Release v1.2.0

**Release Date:** 2026-04-11

## Highlights

### Database Migration: Supabase PostgreSQL to Local SQLite

The project now uses local SQLite database instead of Supabase PostgreSQL. This simplifies setup and removes the dependency on a remote database service.

**Changes:**
- Added `database/schema.sql` - SQLite schema with 6 tables
- Added `database/db.py` - CRUD module for local database
- Updated `scripts/run_analysis.sh` to query SQLite instead of Supabase
- **Auto-initialization:** DB is automatically created on first run if not exists

**Database Tables:**
- `decisions` - Trading decisions
- `portfolio_snapshots` - Portfolio snapshots
- `market_data` - Market data records
- `feedback` - User feedback
- `execution_logs` - Execution logs
- `strategy_history` - Strategy change history

### Telegram Report Format Improvement

- FGI classification now displays in Korean (Extreme Fear -> 극도공포)
- `market` and `portfolio` fields are now structured for better readability

**Before:**
```
108,260,000: KRW
SMA(20) 103,685,200 (+4.41%)
RSI 58.68
FGI 15 (Extreme Fear)
```

**After:**
```
- 현재가: 108,260,000원
- SMA(20): 103,685,200원 (+4.41%)
- RSI: 58.68
- FGI: 15 (극도공포)
```

### Data Cleanup Automation

Added automatic data cleanup to `run_analysis.sh`:

| Directory | Retention | Description |
|-----------|-----------|-------------|
| `data/snapshots/` | 30 days | Execution data snapshots |
| `data/charts/` | 7 days | Captured chart images |
| `logs/executions/` | 30 days | Execution logs |
| `logs/claude_responses/` | 30 days | Claude responses |
